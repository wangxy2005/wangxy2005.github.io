# 256 像素 DDPM 训练效果不佳原因分析

## 1. 现象总结

本次 256x256 像素模型相比之前 128x128 像素模型，生成图像的主观质量明显下降。主要表现为：

- 背景整体偏暗，不如 128 版本干净明亮。
- 背景纹理更杂乱，噪声感和脏污感更明显。
- 背景占比变大，陨石主体有时偏小、不居中或不够稳定。
- 部分图像虽然能生成陨石纹理，但整体构图不如 128 版本自然。

这说明模型并非完全训练失败，而是高分辨率下对“背景”和“主体布局”的建模失衡。

## 2. 主要原因

### 2.1 分辨率提升放大了背景问题

128 像素训练时，原始图片中的背景纹理、阴影、灰尘和边缘噪声会被下采样模糊掉，因此生成结果看起来更干净。

256 像素训练时，图像细节保留更多，背景中的灰暗纹理和杂乱信息也被保留下来。模型在训练中会学习这些背景特征，最终导致生成结果背景更暗、更复杂。

换句话说，256 版本不是单纯学得更清楚，而是把不理想的背景也学得更清楚了。

### 2.2 当前预处理没有真正突出陨石主体

脚本中的 `ResizeWithWhitePadding` 只是将图片等比例缩放后放入白色画布中，并没有裁剪原图中已有的背景，也没有根据陨石主体重新构图。

因此，如果原图中陨石本身占比不大，或者周围有较多暗背景，模型训练时仍然会看到大量背景像素。

在 128 版本中，这个问题被低分辨率缓和了；在 256 版本中，这个问题被明显放大。

### 2.3 全图 MSE 损失会让模型过度关注背景

训练目标使用的是普通全图噪声预测 MSE：

```python
loss = F.mse_loss(noise_pred, noise)
```

对于陨石图片来说，背景通常占据大部分像素，而陨石主体只占一部分区域。全图 MSE 会平等对待所有像素，所以模型优化时会受到大量背景像素影响。

分辨率从 128 提升到 256 后，总像素数变为 4 倍。如果主体占比没有提升，背景像素的主导作用会更明显。结果就是模型更容易学到背景色调和背景纹理，而不是稳定学到陨石主体的形状、位置和比例。

### 2.4 模型结构对 256 分辨率偏弱

当前使用的是 `TinyUNet`，结构较浅，只有 3 次下采样。

对于 128x128 图像，3 次下采样后特征图约为 16x16，模型还能较好地建模全局构图。

对于 256x256 图像，3 次下采样后特征图约为 32x32。虽然通道数从 64 增加到了 96，但模型的全局建模能力并没有同步增强。因此它可以生成局部纹理，却较难稳定控制整个画面的主体大小、位置和背景比例。

这也是为什么 256 版本看起来有局部细节，但整体构图和背景控制更差。

### 2.5 DDIM 采样可能进一步放大观感差异

本次脚本新增了 DDIM 采样，并将默认生成方式设为：

```python
DEFAULT_GENERATE_SAMPLE_STEPS = 500
DEFAULT_SAMPLER = "ddim"
DEFAULT_DDIM_ETA = 0.0
DEFAULT_TIMESTEP_SPACING = "quadratic"
```

DDIM 并不是训练效果变差的根本原因，因为训练阶段仍然是 DDPM 的噪声预测目标。但采样阶段如果使用较少步数、确定性采样和 quadratic timestep spacing，可能会让模型已有的背景偏暗、构图不稳等问题更明显。

因此，128 和 256 的生成效果不应该在采样策略不同的情况下直接比较。若要公平比较，应优先使用相同的 DDPM 1000 steps 采样。

### 2.6 训练轮数增加不一定能解决问题

本次 256 模型已经训练 1300 epochs，比 128 模型的 1200 epochs 更多。但问题主要来自数据预处理、损失权重、模型结构和采样策略，而不是简单的训练轮数不足。

继续训练可能不会让背景变干净，反而可能让模型更牢固地学习到训练集中偏暗、杂乱的背景模式。

## 3. 短期补救方向

由于重新全量训练一次耗时较长，当前阶段更适合从生成端优化。

### 3.1 使用 DDPM 1000 steps 重新生成

为了与 128 像素实验保持一致，建议优先使用完整 DDPM 采样：

```bash
python train_meteorite_ddpm.py generate \
  --checkpoint meteorite_ddpm_output_256/checkpoints/latest.pt \
  --generated-dir generated_pictures_256_ddpm1000 \
  --num-images 1000 \
  --batch-size 16 \
  --sampler ddpm \
  --clean-generated
```

DDPM 1000 steps 通常比 DDIM 500 steps 更稳，尤其是在主体构图和背景一致性方面。

### 3.2 对比不同 checkpoint

最终 checkpoint 不一定是观感最好的。建议分别测试：

- `epoch_0900.pt`
- `epoch_1000.pt`
- `epoch_1100.pt`
- `epoch_1200.pt`
- `epoch_1300.pt`

每个 checkpoint 先生成 32 到 64 张小样本，用肉眼选择背景最干净、主体比例最稳定的版本，再进行全量生成。

### 3.3 如果 DDPM 太慢，使用 DDIM 1000 linear

如果完整 DDPM 生成太慢，可以尝试：

```bash
python train_meteorite_ddpm.py generate \
  --checkpoint meteorite_ddpm_output_256/checkpoints/latest.pt \
  --generated-dir generated_pictures_256_ddim1000_linear \
  --num-images 1000 \
  --batch-size 16 \
  --sampler ddim \
  --sample-steps 1000 \
  --timestep-spacing linear \
  --ddim-eta 0.0 \
  --clean-generated
```

不建议继续把 `DDIM 500 + quadratic` 作为最终生成策略。

### 3.4 超量生成后筛选

如果最终结果中坏图较多，可以生成 1200 到 1500 张，再筛掉：

- 背景明显过暗的图。
- 陨石主体过小的图。
- 主体不清晰或接近空背景的图。
- 背景杂乱程度明显高于其他样本的图。

这比重新训练更节省时间，也更符合当前时间约束。

## 4. 后续重新训练时的改进方向

如果后续有时间重新训练，优先改进顺序建议如下：

1. 改进数据预处理：先检测或裁剪陨石主体，再 resize/pad 到 256，使主体占比更稳定。
2. 使用前景加权损失：让陨石区域在 MSE 中权重更高，减少背景像素主导训练的问题。
3. 加深 UNet：256 分辨率下建议增加第 4 次下采样，使 bottleneck 回到更适合全局建模的尺度。
4. 加入 attention：在 32x32 或 16x16 特征层加入 self-attention，增强主体位置和整体构图能力。
5. 统一采样策略：比较不同实验时，采样方式、步数和 timestep spacing 应保持一致。
6. 尝试从 128 模型迁移：如果模型通道数保持一致，可以用 128 模型作为初始化，再 finetune 到 256。

## 5. 结论

256 像素训练效果不佳的主要原因不是训练时间太短，而是分辨率提升后，原本数据中的背景问题被放大了。同时，当前 TinyUNet 结构和全图 MSE 损失更容易让模型学习背景，而不是稳定学习陨石主体。

短期内最实际的优化方案是：使用相同口径的 DDPM 1000 steps 重新生成，并对不同 checkpoint 做小样本对比。长期来看，应优先改进主体裁剪、前景权重和 256 分辨率下的模型结构。
